Windows Setup

Setup for Windows Host (10, 11, and Snapdragon)

Follow these instructions to install the Qualcomm AI Engine Direct SDK (commonly referred to as the QNN SDK) and all necessary dependencies.

Note

This guide is for a host machine running Windows 10, 11, or Windows on Snapdragon. If you are using a Linux machine or are using a WSL environment, follow the instructions here.

The QNN SDK allows you to convert an AI model (ex. a .pt model from PyTorch) into instructions that can be run on a target device’s various processing units (CPU, GPU, HTP, cDSP, or LPAI).

Note

To learn about acronyms which are new to you, see the Glossary.

You will need to install the following code to use the QNN SDK:

  1. The QNN SDK binary and its immediate dependencies.

  2. ML frameworks to interpret your AI model files. (Ex. PyTorch)

  3. Toolchain code for your target device’s operating system.

    1. Ex. To develop on android, you will need to install an Android NDK.

  4. Packages/tooling to build the converted models for specific hardware processors (CPU, GPU, HTP, cDSP, or LPAI).

Note

This guide contains many recommendations about specific versions of code to use that have been verified to work. Other versions of those dependencies may or may not work, so use them at your own risk.

1. Install Qualcomm AI Engine Direct (aka the “QNN SDK”)

  1. Go to the QNN SDK product page: https://www.qualcomm.com/developer/software/qualcomm-ai-engine-direct-sdk

  2. Click “Get Software” to download the QNN SDK

    1. Note: The QNN SDK is ~2 Gb when unzipped.

  3. Unzip the downloaded SDK.

    1. The zipped file should have a name like v2.22.6.240515.

    2. When unzipped, the folder should be named qairt with a sub-folder that looks like 2.22.6.240515.

Set up your environment

  1. Open an admin powershell terminal by:

    1. Pressing the “Windows” button to search for applications

    2. Searching for “Powershell”

    3. Right clicking “Windows Powershell” and selecting “Run as Administrator”

    4. Saying “Yes” when prompted for permission to run this application as an administrator.

  2. Navigate to qairt/<QNN_SDK_VERSION> inside the unzipped QNN SDK.

    1. Replace <QNN_SDK_VERSION> with the name of the file directly under qairt - it should look something like 2.22.6.240515.

  3. Run cd ./bin.

  4. Run Unblock-File ./envsetup.ps1.

    This will allow envsetup.ps1 to be run on your computer.

  5. Run ./envsetup.ps1.

    This will automatically set the environment variables QNN_SDK_ROOT and SNPE_ROOT.

    1. QNN_SDK_ROOT - This points to the qairt/QNN_SDK_ROOT folder where the QNN SDK code lives.

    2. SNPE_ROOT - This stands for Snapdragon Neural Processing Engine (a synonym for Qualcomm® Neural Processing SDK). This SDK is included as part of the QNN SDK, so the variable points to the same folder as QNN_SDK_ROOT.

Warning

The envsetup.ps1 script only updates environment variables for this powershell instance. If you need to set QNN_SDK_ROOT again, just re-run this script.

  1. Run ``Unblock-File “${QNN_SDK_ROOT}/bin/check-windows-dependency.ps1” ``.

  2. Run ``& “${QNN_SDK_ROOT}/bin/check-windows-dependency.ps1” ``

    This will install the core Windows build tools.

    1. Each time you run the script it will install more dependencies. You will likely have to re-run the command after it helps you install VS Studio and VS Studio Build Tools.

    2. When you have installed all the necessary dependencies, the script will say “All Done”.

  3. Run ``Unblock-File “${QNN_SDK_ROOT}/bin/envcheck.ps1” ``.

  4. Run & "${QNN_SDK_ROOT}/bin/envcheck.ps1" -m.

    This will verify that you have installed the Microsoft Visual Studio C++ (MSVC) toolchain successfully.

Note

It’s normal to see that only one of “Visual C++(x86)” and “Visual C++(arm64)” is installed. The installed version should match the hardware of your host machine.

2. Install QNN SDK dependencies

  1. Install Python 3.10.4 - http://python.org/downloads/release/python-3104/

Warning

Ensure you are downloading the proper version. New releases beyond Python 3.10.4 may not work as expected with the QNN SDK.

  1. Verify the installation worked by running:

py --list

You should see -V:3.10 as one of the options.

  1. Open the folder at QNN_SDK_ROOT in Visual Studio.

  2. Open a new powershell terminal in Visual Studio by going to the top-menu “Terminal” and clicking “New Terminal”

Warning

Ensure the new terminal is a Powershell instance.

  1. From the QNN_SDK_ROOT folder, run the following command to create and activate a new virtual environment:

py -3.10 -m venv "venv"
& "venv\Scripts\Activate.s1"

  1. Update all dependencies by running the following commands in Powershell:

python -m pip install --upgrade pip
python "$env:QNN_SDK_ROOT\bin\check-python-dependency"

3. Install Model Frameworks

These are the packages that interpret your AI model files. For example, the PyTorch package interprets .pt files.

Note

Install any / all of the frameworks you want to use AI models from.
You can install a package by running pip install package==version.
Ex. pip install tensorflow==2.10.1
